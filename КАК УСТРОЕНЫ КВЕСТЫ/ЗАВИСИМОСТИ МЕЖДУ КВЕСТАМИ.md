# Зависимости между квестами

Квест можно заблокировать до выполнения других квестов. Зависимости задаются в поле `"after"` датапак-квеста списком **групп**:

```json
{
	"after": [
		["namespace:quest_a", "namespace:quest_b"],
		["namespace:quest_c"]
	]
}
```

Квест разблокируется, когда **все квесты хотя бы в одной группе** успешно завершены. Это AND внутри группы, OR между группами.

**Примеры:**
- `[["a", "b"]]` — квест разблокируется только если выполнены и A, и B.
- `[["a"], ["b"]]` — квест разблокируется если выполнен A или B.
- `[["a", "b"], ["c"]]` — квест разблокируется если выполнены (A и B) или C.

Зависимости — декларативная конфигурация для управления прогрессией. Проверка зависимостей не блокирует выдачу квеста командой `quest give`: оператор всегда может выдать квест вручную независимо от того, выполнены ли зависимости.

## Дополнительные условия разблокировки (`require`)

Помимо зависимостей по квестам, разблокировку можно ограничить дополнительными условиями через поле `"require"`. Это удобно, когда квест должен открыться только при определённых обстоятельствах — например, если у игрока есть особый тег или выполняется predicate.

```json
{
	"after": [
		["namespace:prologue"]
	],
	"require": {
		"tags": ["found_secret_passage"],
		"predicate": "namespace:is_nighttime"
	}
}
```

- `require.tags` — список scoreboard-тегов, которые должны быть у игрока в момент завершения зависимости.
- `require.predicate` — идентификатор predicate датапака, который оценивается против игрока.

Оба поля необязательные. `require` без `after` игнорируется — условия разблокировки проверяются только в момент завершения зависимого квеста.

**Пример с только тегом:**
```json
{
	"after": [["story:ch1"]],
	"require": {
		"tags": ["chose_dark_path"]
	}
}
```

Квест откроется только игрокам, у которых есть тег `chose_dark_path` — то есть тем, кто выбрал тёмный путь в предыдущей главе.
