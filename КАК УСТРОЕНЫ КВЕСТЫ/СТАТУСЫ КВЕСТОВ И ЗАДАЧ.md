# Статусы квестов и задач

Когда квест выдаётся игроку — он становится **отслеживаемым** (`tracked`). До выполнения отслеживаемый квест считается **активным** (`active`). После завершения — **завершённым** (`complete`). Завершённые квесты всё ещё считаются `tracked`, но их прогресс больше не обновляется.

Иерархия состояний квеста:

```
tracked
├── active
│   └── pinned  (закреплён в HUD)
└── complete
    ├── success
    ├── failure
    └── skipped
```

Задачи имеют аналогичный набор статусов. Задача считается `active`, если она находится в активном этапе и ещё не выполнена. Задача `pinned` — выбранная для отображения в HUD (одна на квест). Когда задача выполняется — она становится `complete` с соответствующим статусом успеха/провала/пропуска.

Проверить статусы можно командами `quest test` и `/execute if quest`.

> Квест без этапов остаётся `active` бесконечно — у него нет обязательных задач, по которым можно вычислить итоговый статус. Такие квесты применяются для нарративных записей (см. [[КВЕСТ-ЗАМЕТКА]]).
