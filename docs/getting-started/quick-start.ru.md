# Быстрый старт

Соберём первый квест — «Побег» с основным заданием «Найти и опустить рычаг» и дополнительным «Узнать что стало с выжившими». Сделать это можно двумя путями, и квест здесь — не просто галочка в списке: он живёт, проходит этапы и может завершиться успехом, провалом или быть пропущен.

- **Командами** — быстро попробовать прямо в игре, не создавая файлов вручную. Такой квест (динамический) сохраняется внутри мира и существует только вместе с ним.
- **Через датапак** — когда квест станет частью контента. Датапак можно положить в мир, но при необходимости отцепить и распространять отдельно — как самостоятельный набор квестов.

Оба пути дают один и тот же квест. Начните с команд, если просто хотите пощупать систему.

---

## Путь 1. Квест командами

Создаём квест и две его задачи — обязательную (`required`) и дополнительную (`optional`):

```
/quest new example:escape "Побег" "Здесь больше нельзя оставаться, но как же выбраться?"
/quest modify example:escape tasks add required push_the_lever "Найти и опустить рычаг открывающий люк" "Когда один из стражей выходил, мне удалось услышать щелчок в дальней части комнаты"
/quest modify example:escape tasks add optional look_for_survivors "Узнать что стало с выжившими" "Нужно обыскать камеры, может удастся найти ещё кого-нибудь, кому удалось спастись"
```

Выдаём квест игроку и закрепляем его в HUD:

```
/quest give PlayerName example:escape
/quest pin PlayerName example:escape
```

После выдачи игрок увидит подсказку о новом квесте, а закреплённый квест появится в HUD на экране. Полный список квестов открывается клавишей `J` — подробнее в [ИНТЕРФЕЙС ИГРОКА](/player-view/player-interface.md).

Чтобы завершить задачи, можно спрятать на карте командные блоки с такими командами:

```
/quest complete PlayerName example:escape task push_the_lever success
/quest complete PlayerName example:escape task look_for_survivors success
```

> Задачи, добавленные командой, завершаются только командой `quest complete`. Чтобы задача срабатывала сама — по scoreboard, predicate или таймеру — нужен датапак-квест (см. ниже и [УСЛОВИЯ ВЫПОЛНЕНИЯ ЗАДАЧИ](/datapack-reference/task-completion-conditions.md)).

## Путь 2. Тот же квест в датапаке

Статический квест описывается в JSON, версионируется и распространяется в составе датапака — отдельно от мира или вместе с ним. В отличие от командного, он поддерживает автозавершение задач. Положите его в датапак мира:

```
<мир>/datapacks/example/
├── pack.mcmeta
└── data/example/quests/escape.json
```

`pack.mcmeta` — служебный файл датапака (`pack_format: 15` соответствует Minecraft 1.20.1):

```json
{
	"pack": {
		"pack_format": 15,
		"description": "Example quests"
	}
}
```

`escape.json` — сам квест. Имя файла (`escape`) и папка (`example`) вместе дают идентификатор `example:escape`:

```json
{
	"version": 3,
	"variant": 1,
	"title": "Побег",
	"description": "Здесь больше нельзя оставаться, но как же выбраться?",
	"tasks": {
		"push_the_lever": {
			"title": "Найти и опустить рычаг открывающий люк",
			"description": "Когда один из стражей выходил, мне удалось услышать щелчок в дальней части комнаты"
		},
		"look_for_survivors": {
			"title": "Узнать что стало с выжившими",
			"description": "Нужно обыскать камеры, может удастся найти ещё кого-нибудь, кому удалось спастись"
		}
	},
	"stages": [
		["push_the_lever", "look_for_survivors"]
	]
}
```

Несколько вещей о структуре:

- `version` и `variant` — обязательные. Они защищают от загрузки в несовместимой версии мода; текущие значения — `version: 3`, `variant: 1`. Разбор — в [ФОРМАТ ФАЙЛА КВЕСТА](/datapack-reference/quest-file-format.md).
- `stages` — это список этапов, каждый этап — список ID задач. **Первая задача в этапе обязательная, остальные — дополнительные.** Здесь один этап: `push_the_lever` ведёт прогресс, `look_for_survivors` идёт сбоку.

Загрузите датапак командой `/reload` и убедитесь, что квест считался:

```
/quest list static
```

Если `example:escape` есть в списке — всё готово, дальше квест выдаётся той же командой `quest give`. Если его нет — загляните в лог сервера и в [ЧАСТЫЕ ВОПРОСЫ И ОШИБКИ](/troubleshooting/common-questions-and-errors.md).

> Не всякий квест должен быть «заданием с прогрессом». Можно сделать квест-заметку без задач — страницу дневника, слух, фрагмент лора — или ивент с общим прогрессом. Готовые рецепты — в [ПАТТЕРНЫ КВЕСТОВ](/quest-patterns/quest-patterns.md).
