# Структура квеста

Квест представляет из себя набор задач, поделённых на этапы. Каждый этап имеет ровно одну обязательную задачу и любое число необязательных. Этапы идут последовательно — только после завершения обязательной задачи текущего этапа открывается следующий.

**Исходы квеста:**
- **Успех** (`success`) — все обязательные задачи выполнены успешно или пропущены, но не все пропущены.
- **Провал** (`failure`) — хотя бы одна обязательная задача провалена. Квест завершается немедленно.
- **Пропуск** (`skipped`) — все обязательные задачи пропущены.

Задача завершается при выполнении её условия успеха (или условия провала), через кнопку в квестовой книге (если задача настроена с `buttons`) или явной командой:

```
/quest complete <player> <quest> task <taskId> skip
```

Квест завершается в тот же тик, когда завершается его последняя незавершённая обязательная задача.

> Все поля файла квеста и задачи — с типами и значениями по умолчанию — собраны в [ФОРМАТ ФАЙЛА КВЕСТА](/datapack-reference/quest-file-format.md). Там же разобрано, какие формы текста принимают `title` и `description` и какие стили работают.

## Квесты без этапов

Этапы и задачи — необязательны. Квест может быть полностью без `stages` (пустой массив или пропущенное поле) и без задач. Такой квест:

- появляется в книге как `active` после `quest give`;
- не показывается в HUD (нечего закреплять);
- **никогда не завершается автоматически** — у него нет ни одной обязательной задачи, по которой система могла бы рассчитать исход.

Это полезно для нарративных записей — слухов, заметок, фрагментов лора, — которые должны существовать в дневнике игрока без требования что-то делать. Подробно паттерн разобран в [Паттернах: квест-заметка](/quest-patterns/quest-note.md).
