# Повторяемые квесты

Обычный квест проходится один раз: после завершения его нельзя выдать тому же игроку снова. Повторяемый квест снимает это ограничение — завершённый квест можно выдать заново сколько угодно раз. Это основа ежедневных заданий, фарм-квестов и повторяющихся ивентов.

Квест становится повторяемым через флаг `repeatable`:

```json
{
	"version": 3,
	"variant": 1,
	"title": "Дневной сбор древесины",
	"repeatable": true
}
```

По умолчанию `repeatable: false` — квест одноразовый.

---

## Как это работает

Когда вы выдаёте повторяемый квест игроку, который его **уже завершил** (командой `quest give` или через [свиток](/player-view/items.md)), система автоматически сбрасывает прогресс прошлого прохождения перед новой выдачей. Игрок начинает квест заново с чистого листа — задачи снова не выполнены, активный этап снова первый.

Ключевая граница: повторно выдать можно только **завершённый** квест.

| Состояние квеста у игрока | `repeatable: false` | `repeatable: true` |
|---|---|---|
| Не выдан | Выдаётся | Выдаётся |
| Активен (не завершён) | Повторная выдача запрещена | Повторная выдача запрещена |
| Завершён (успех/провал/пропуск) | Повторная выдача запрещена | Выдаётся заново, прогресс сбрасывается |

Активный квест нельзя перевыдать ни с каким значением `repeatable` — сначала его нужно завершить или снять командой `quest drop`.

> При новой выдаче повторяемого квеста создаётся новая «жизнь» его задач: хук `on.load` сработает заново. Подробнее о жизни задачи — в [ХУКИ ЖИЗНЕННОГО ЦИКЛА ЗАДАЧИ](/datapack-reference/task-lifecycle-hooks.md).

## Включение через команду

Для динамических квестов флаг можно переключать на лету:

```
/quest modify example:daily_quest repeatable true
```

Команда работает только с динамическими квестами; у статических флаг задаётся в JSON. См. [СПРАВОЧНИК КОМАНД](/command-api/command-reference.md#quest-modify--repeatable).

---

## Сброс по расписанию

Сам по себе `repeatable` не «обновляет» квест по таймеру — он лишь разрешает повторную выдачу. Чтобы получить настоящую ежедневку, повторную выдачу запускает функция датапака по своему расписанию (например, по игровому времени суток). Полный рецепт с готовым JSON и mcfunction — в паттерне [ПОВТОРЯЕМЫЙ КВЕСТ](/quest-patterns/repeatable-quest.md).
