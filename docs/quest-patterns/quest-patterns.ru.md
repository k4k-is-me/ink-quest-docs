# Паттерны квестов

## Что такое «паттерн»

В этом разделе **паттерн** — это рецепт, как собрать знакомую нарративную или геймплейную механику из примитивов InkQuest. Сами по себе паттерны — не фича системы и не специальный тип квеста. Это **готовые комбинации** обычных полей JSON: этапов, задач, условий, хуков и scoreboard-команд.

InkQuest сознательно не вводит специальные «типы квестов» вроде «таймер», «эскорт» или «развилка». Вместо этого даются примитивы, из которых эти типы собираются. Поэтому документация не описывает «как включить таймер» — она описывает, **как сложить таймер своими руками**, и то же самое с эскортом, развилкой, ежедневкой, серверным ивентом.

### Зачем читать паттерны

- **Стартовая библиотека.** Можно не изобретать с нуля — взять готовый JSON и подогнать под себя.
- **Объяснение «почему так».** Каждый паттерн показывает не только сборку, но и подводные камни, которые без подсказки наступишь.
- **Граница системы.** Понимая, что любая нестандартная механика собирается из тех же примитивов — становится ясно, чего система **не** делает и что придётся подать через датапак-функции, advancement'ы или вызовы команд.

### Чем паттерн отличается от справочника

[СПРАВОЧНИК КОМАНД](/command-api/command-reference.md) и [СТРУКТУРА КВЕСТА](/how-quests-work/quest-structure.md) отвечают на вопрос **«что есть в системе»**. Паттерны отвечают на вопрос **«как этим воспользоваться, чтобы получить конкретный эффект»**. Если справочник — это словарь, то паттерны — это фразы.

## Содержание

Порядок — по сложности и логике обучения. Если читаешь впервые — иди сверху вниз.

1. **[КВЕСТ-ЗАМЕТКА](/quest-patterns/quest-note.md)** — самый простой паттерн: квест без задач, как страница дневника.
2. **[ВЫДАЧА КВЕСТОВ НОВОМУ ИГРОКУ](/quest-patterns/granting-quests-to-new-player.md)** — как раздать стартовый квест автоматически при первом подключении, без команд оператора.
3. **[ЦЕПОЧКА КВЕСТОВ](/quest-patterns/quest-chain.md)** — как связать квесты, чтобы один открывал следующий.
4. **[ОЖИДАНИЕ СОБЫТИЯ](/quest-patterns/waiting-for-an-event.md)** — как сделать паузу между квестами (по времени, по событию).
5. **[ЗАДАЧА С ДВУМЯ ИСХОДАМИ](/quest-patterns/task-with-two-outcomes.md)** — мощный нарративный приём: одно неверное действие — тёмная ветка истории.
6. **[РАЗВИЛКА И ПАМЯТЬ ВЫБОРА](/quest-patterns/branch-and-choice-memory.md)** — нелинейный сюжет: игрок выбирает, последующие квесты подстраиваются.
7. **[ПОВТОРЯЕМЫЙ КВЕСТ](/quest-patterns/repeatable-quest.md)** — квест, который можно проходить много раз (ежедневки, гринд).
8. **[ГЛОБАЛЬНЫЙ КВЕСТ](/quest-patterns/global-quest.md)** — серверный ивент с общим прогрессом всех игроков.

## Краткий справочник по применимости

| Паттерн | Применять, когда… |
| --- | --- |
| [КВЕСТ-ЗАМЕТКА](/quest-patterns/quest-note.md) | Нужна нарративная запись в дневнике без задач (слух, лор, письмо) |
| [ВЫДАЧА КВЕСТОВ НОВОМУ ИГРОКУ](/quest-patterns/granting-quests-to-new-player.md) | Новый игрок должен сразу получить стартовый квест без участия оператора |
| [ЦЕПОЧКА КВЕСТОВ](/quest-patterns/quest-chain.md) | Сюжет идёт линейно: квест A → квест B → квест C |
| [ОЖИДАНИЕ СОБЫТИЯ](/quest-patterns/waiting-for-an-event.md) | Следующий квест должен открыться не сразу, а по времени или событию |
| [ЗАДАЧА С ДВУМЯ ИСХОДАМИ](/quest-patterns/task-with-two-outcomes.md) | Игрок может либо выполнить задачу, либо безвозвратно её провалить |
| [РАЗВИЛКА И ПАМЯТЬ ВЫБОРА](/quest-patterns/branch-and-choice-memory.md) | У игрока есть выбор, и этот выбор влияет на дальнейший сюжет |
| [ПОВТОРЯЕМЫЙ КВЕСТ](/quest-patterns/repeatable-quest.md) | Квест должен проходиться многократно (ежедневки, фарм-задания) |
| [ГЛОБАЛЬНЫЙ КВЕСТ](/quest-patterns/global-quest.md) | Все игроки сервера вместе работают на один прогресс |

## Структура каждой страницы

Все паттерны написаны по одному шаблону, рассчитанному на постепенное погружение:

1. **Лид и «Зачем нужен этот паттерн»** — короткое объяснение, чтобы понять, твой ли это случай.
2. **Шаги 1, 2, 3…** — минимальный рабочий пример с готовым JSON и mcfunction.
3. **Проверка** — чек-лист «выдай → выполни → убедись».
4. **Расширения** — варианты усложнения от частого к редкому. Можно пропустить и вернуться, когда понадобится.
5. **Подводные камни** — типичные ошибки с объяснением «почему».
6. **См. также** — ссылки на смежные паттерны и базовые разделы.

Не обязательно читать всё подряд — можно дойти до «Шага 1», скопировать JSON и доделать остальное по мере необходимости.

## Эти паттерны — не закрытый список

Любая нестандартная механика собирается из тех же примитивов: задачи, [условия](/datapack-reference/task-completion-conditions.md), [хуки](/datapack-reference/task-lifecycle-hooks.md) и [команды](/command-api/command-reference.md). Если у тебя есть свой нарративный приём, который не вписывается ни в один из готовых — скорее всего, он реализуется без новых полей в JSON. Если нет — это повод обсудить расширение системы.
