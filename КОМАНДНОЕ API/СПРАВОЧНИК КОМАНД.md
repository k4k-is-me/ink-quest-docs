# Справочник команд

Все команды требуют уровень оператора 2+. Аргументы в угловых скобках `<...>` — обязательные, в квадратных `[...]` — необязательные.

**Типы аргументов:**
- `Selector` — селектор игрока (`@s`, `@a`, `PlayerName`). Все команды принимают только одного игрока (не `@a`).
- `Identifier` — идентификатор вида `namespace:name` (например `example:my_quest`)
- `taskId` — строка-идентификатор задачи внутри квеста (без namespace)
- `Text` — JSON-текст или строка в кавычках

## Создание и редактирование квестов

> Эти команды работают только с **динамическими** квестами (созданными командой). Квесты из датапака (`static`) редактировать через команды нельзя.

---

### `quest new`

```
/quest new <id: Identifier> [<title: Text>] [<description: Text>]
```

Создаёт новый динамический квест. Title и description можно задать сразу или позже через `quest modify`.

**Для чего полезна:** создание квестов "на лету" без правки датапака — для контента, зависящего от состояния мира, ивентов или конкретных игроков.

**Примеры:**

- *Практичный:* ежедневный квест, создаваемый функцией автоматизации:
  ```
  /quest new server:daily_hunt "Дневная охота" "Задание обновляется каждый день"
  ```
- *Креативный:* уникальный квест под конкретного игрока, сгенерированный функцией по его имени:
  ```
  /quest new generated:player_123_mystery "Тайна незнакомца"
  ```

**Ошибки:**
- Квест с таким идентификатором уже существует

---

### `quest modify ... title`

```
/quest modify <id: Identifier> title <title: Text>
```

Изменяет название квеста.

**Для чего полезна:** правка текстов без пересоздания квеста — особенно удобно для ивентов, где название меняется по ходу события.

**Примеры:**

- *Практичный:* обновить название за день до конца ивента:
  ```
  /quest modify event:hunt title "Охота: финальный день"
  ```
- *Креативный:* изменить название как нарративное событие, когда игрок что-то открыл:
  ```
  /quest modify story:prophecy title "Пророчество о тебе"
  ```

---

### `quest modify ... description`

```
/quest modify <id: Identifier> description <description: Text>
```

Изменяет описание квеста.

**Для чего полезна:** обновление контекста или подсказок в квесте по мере развития нарратива.

**Примеры:**

- *Практичный:* дополнить описание подсказкой после того, как игрок застрял:
  ```
  /quest modify story:dungeon description "Говорят, кузнец знает о тайном ходе..."
  ```
- *Креативный:* изменить описание как реакцию на действие игрока — "пророчество исполнилось":
  ```
  /quest modify story:prophecy description "Пророчество исполнилось. Что-то изменилось в мире."
  ```

---

### `quest modify ... tasks add`

```
/quest modify <id: Identifier> tasks add required <taskId: string> [<title: Text>] [<description: Text>]
/quest modify <id: Identifier> tasks add optional <taskId: string> [<title: Text>] [<description: Text>]
```

Добавляет задачу к квесту.

- `required` — создаёт новый этап с этой задачей как обязательной.
- `optional` — добавляет задачу в последний существующий этап как необязательную. Требует, чтобы в квесте уже был хотя бы один этап (т.е. хотя бы одна `required` задача).

Задачи, добавленные командой, не имеют условий выполнения — их можно выполнить только командой `quest complete`.

**Для чего полезна:** структурирование квеста этапами и боковыми ответвлениями прямо в игре.

**Примеры:**

- *Практичный:* добавить три последовательных этапа туториала:
  ```
  /quest modify tutorial:start tasks add required gather "Собери ресурсы"
  /quest modify tutorial:start tasks add required craft "Скрафти инструмент"
  /quest modify tutorial:start tasks add required explore "Исследуй пещеру"
  ```
- *Креативный:* добавить секретную опциональную задачу в уже существующий этап:
  ```
  /quest modify story:ch1 tasks add optional secret_room "Найди тайную комнату" "Что-то скрипит за водопадом..."
  ```

**Ошибки:**
- Задача с таким `taskId` уже существует в этом квесте
- Попытка добавить `optional` задачу в квест без этапов

---

### `quest modify ... tasks remove`

```
/quest modify <id: Identifier> tasks remove <taskId: string>
```

Удаляет задачу из квеста.

**Для чего полезна:** реструктуризация квеста — удаление устаревших или багованных задач.

**Примеры:**

- *Практичный:* убрать отладочную задачу перед релизом:
  ```
  /quest modify tutorial:start tasks remove debug_skip
  ```
- *Креативный:* убрать альтернативный путь после нарративного события — развилка "исчезла":
  ```
  /quest modify story:ch2 tasks remove locked_path
  ```

**Ограничение:** нельзя удалить задачу из квеста, который хотя бы один игрок отслеживает (`tracked`). Сначала нужно снять квест со всех игроков (`quest drop` или `quest purge`).

---

### `quest modify ... icon`

```
/quest modify <id: Identifier> icon <icon: Identifier>
```

Устанавливает иконку квеста (предмет из реестра Minecraft, например `minecraft:book`).

**Для чего полезна:** визуальная идентификация квеста в книге — иконка сразу передаёт тематику квеста.

**Примеры:**

- *Практичный:* задать книгу как иконку лорного квеста:
  ```
  /quest modify story:prologue icon minecraft:written_book
  ```
- *Креативный:* иконка намекает на яд, не раскрывая сюжет детективного квеста:
  ```
  /quest modify mystery:killer icon minecraft:suspicious_stew
  ```

---

### `quest modify ... index`

```
/quest modify <id: Identifier> index <n: int>
```

Устанавливает порядковый индекс квеста для сортировки в книге квестов. Квесты с меньшим индексом отображаются выше.

**Для чего полезна:** управление порядком квестов в книге — туториалы первыми, сайд-квесты в конце.

**Примеры:**

- *Практичный:* туториал всегда в начале списка:
  ```
  /quest modify tutorial:basics index 0
  ```
- *Креативный:* "спрятать" секретный квест в самый конец списка, чтобы его было сложнее заметить:
  ```
  /quest modify secret:easter_egg index 9999
  ```

---

### `quest modify ... repeatable`

```
/quest modify <id: Identifier> repeatable true|false
```

Включает или отключает возможность повторного прохождения квеста. При `true` завершённый квест можно выдать снова — прогресс предыдущего прохождения сбрасывается автоматически. При `false` (по умолчанию) выдача уже завершённого квеста блокируется.

> Активный квест (ещё не завершённый) нельзя перевыдать ни с каким значением repeatable.

**Для чего полезна:** ежедневные задания, фарм-квесты, ивентовые механики, которые должны работать повторно.

**Примеры:**

- *Практичный:* дейлик, который можно выполнять каждый день:
  ```
  /quest modify daily:wood_gathering repeatable true
  ```
- *Креативный:* испытание на выживание — игрок может повторять его сколько угодно раз:
  ```
  /quest modify challenge:survival_run repeatable true
  ```

**Ошибки:**
- `Quest is static` — попытка изменить квест из датапака (только динамические).

---

### `quest modify ... pin_mode`

```
/quest modify <id: Identifier> pin_mode auto|off|force
```

Устанавливает режим автозакрепления квеста, который применяется при его автоматической выдаче — когда квест выдаётся как зависимый от только что завершённого.

- `auto` — закрепить, если родительский квест был закреплён в HUD
- `force` — закрепить всегда
- `off` — не закреплять автоматически

> Этот режим не влияет на ручную выдачу командой `quest give`.

**Для чего полезна:** управление тем, как квест "входит" в HUD при автовыдаче по цепочке зависимостей.

**Примеры:**

- *Практичный:* туториальный квест всегда виден новичку:
  ```
  /quest modify tutorial:basics pin_mode force
  ```
- *Креативный:* финальный квест намеренно не всплывает в HUD — игрок должен открыть книгу и найти его сам:
  ```
  /quest modify epilogue:credits pin_mode off
  ```

---

### `quest remove`

```
/quest remove <id: Identifier>
```

Удаляет динамический квест из системы. После удаления квест перестаёт существовать — его нельзя выдать игрокам или модифицировать.

**Для чего полезна:** зачистка временных квестов — тестовых или завершившихся ивентов.

**Примеры:**

- *Практичный:* удалить тестовый квест после отладки:
  ```
  /quest remove dev:test_01
  ```
- *Креативный:* убрать сезонный ивент-квест по окончании события (после `quest purge`):
  ```
  /quest remove event:new_year_2025
  ```

**Ограничение:** нельзя удалить квест, который хотя бы один игрок отслеживает. Сначала нужно снять квест со всех игроков (`quest drop` или `quest purge`).

**Ошибки:**
- Квест не существует
- Квест является статическим (из датапака)
- Квест отслеживается хотя бы одним игроком

---

## Управление квестами игрока

---

### `quest give`

```
/quest give <player: Selector> <id: Identifier>
/quest give <player: Selector> <id: Identifier> pin
```

Выдаёт квест игроку. После этого квест становится `tracked`. Если указан флаг `pin` — квест также закрепится в HUD.

**Для чего полезна:** точка входа игрока в любой нарратив — выдаётся из функций, командных блоков, триггеров.

**Примеры:**

- *Практичный:* выдать туториал с закреплением при первом входе на сервер (через `load` функцию):
  ```
  /quest give @s tutorial:basics pin
  ```
- *Креативный:* выдать квест в момент взаимодействия с NPC-табличкой (командный блок с `/execute as @p[distance=..2]`):
  ```
  /execute as @p[distance=..2] run quest give @s story:rescue_blacksmith pin
  ```

**Ошибки:**
- Квест не существует
- Игрок уже отслеживает этот квест

---

### `quest drop`

```
/quest drop <player: Selector> <id: Identifier>
```

Забирает квест у игрока. Весь прогресс по квесту теряется. Если квест был закреплён в HUD — открепляется.

**Для чего полезна:** сброс прогресса — при баге, перезапуске сюжетной линии или как нарративное событие.

**Примеры:**

- *Практичный:* сбросить квест игроку, застрявшему из-за бага:
  ```
  /quest drop PlayerName story:ch1
  ```
- *Креативный:* нарративное событие "предатель забрал задание" — срабатывает из хука `success` задачи-провокации:
  ```
  /quest drop @s story:diplomatic_mission
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает этот квест

---

### `quest complete ... task`

```
/quest complete <player: Selector> <id: Identifier> task <taskId: string> success|failure|skip
```

Выполняет конкретную задачу квеста с указанным статусом.

- Позволяет выполнять задачи из любого этапа, не только активного.
- Если выполненная задача является обязательной и все условия перехода соблюдены — квест перейдёт на следующий этап или завершится.

**Для чего полезна:** ручное управление отдельными задачами — для модерации, нарративных развилок или обхода технических проблем.

**Примеры:**

- *Практичный:* вручную засчитать задачу "поговори с NPC" для игрока, у которого сломался триггер:
  ```
  /quest complete Kuchukov story:ch1 task talk_to_blacksmith success
  ```
- *Креативный:* пропустить задачу как нарративный выбор — игрок "уступил", и квест продолжается альтернативной веткой:
  ```
  /quest complete @s story:standoff task defend_gate skip
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест уже завершён
- Задача уже выполнена

---

### `quest complete ... stage`

```
/quest complete <player: Selector> <id: Identifier> stage success|failure|skip full|required
```

Выполняет задачи активного этапа квеста.

- `required` — выполняет только обязательную задачу этапа.
- `full` — выполняет все задачи этапа (обязательную и все необязательные).

Возвращает количество выполненных задач.

**Для чего полезна:** массовое завершение этапа — удобно для пропуска обучения или нарративных переходов.

**Примеры:**

- *Практичный:* опытный игрок пропускает туториальный этап:
  ```
  /quest complete @s tutorial:basics stage skip full
  ```
- *Креативный:* нарративное событие "орк прорвал оборону" — этап проваливается, квест переходит к "плохой" ветке:
  ```
  /quest complete @s story:defend_village stage failure required
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест уже завершён
- У квеста нет активного этапа

---

### `quest complete` (весь квест)

```
/quest complete <player: Selector> <id: Identifier> success|failure|skip full|required
```

Выполняет все незавершённые задачи квеста.

- `required` — выполняет только обязательные задачи всех этапов.
- `full` — выполняет все задачи всех этапов.

Возвращает количество выполненных задач.

**Для чего полезна:** быстрый пропуск целой сюжетной главы — для тестирования, модерации или наказания.

**Примеры:**

- *Практичный:* пропустить первую главу для тестирования второй:
  ```
  /quest complete @s story:ch1 success required
  ```
- *Креативный:* система обнаружила читерство — квест завершается провалом как наказание:
  ```
  /quest complete cheater story:hero failure required
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест уже завершён

---

## Закрепление в HUD

---

### `quest pin`

```
/quest pin <player: Selector> <id: Identifier>
/quest pin <player: Selector> <id: Identifier> <taskId: string>
```

Закрепляет квест в HUD игрока.

- Без `taskId` — закрепляет обязательную задачу активного этапа.
- С `taskId` — закрепляет указанную задачу. Задача должна быть активной (находиться в активном этапе и не быть выполненной).

**Для чего полезна:** управление тем, что игрок видит в HUD — переключение между задачами или программное закрепление после кат-сцены.

**Примеры:**

- *Практичный:* закрепить конкретную опциональную задачу вместо обязательной (по умолчанию закрепилась бы обязательная):
  ```
  /quest pin @s story:dungeon treasure_hint
  ```
- *Креативный:* через хук `success` задачи переключить закреплённый квест на следующую главу:
  ```
  /quest unpin @s story:ch1
  /quest pin @s story:ch2
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест уже закреплён (первая форма)
- Задача не активна (вторая форма)
- Задача уже закреплена (вторая форма)

---

### `quest unpin`

```
/quest unpin <player: Selector> <id: Identifier>
```

Открепляет квест из HUD игрока.

**Для чего полезна:** убрать квест из HUD на время кат-сцены, финального боя или при программном переключении.

**Примеры:**

- *Практичный:* открепить квест по окончании обучающей кат-сцены:
  ```
  /quest unpin @s tutorial:basics
  ```
- *Креативный:* убрать квест из HUD на время финального босс-файта, чтобы не мешал интерфейс:
  ```
  /quest unpin @s story:main_quest
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест не закреплён

---

## Просмотр информации

---

### `quest list`

```
/quest list
/quest list all
/quest list static
/quest list dynamic
```

Выводит список всех зарегистрированных квестов (загруженных из датапаков и/или созданных командами).

- `all` (по умолчанию) — все квесты
- `static` — только квесты из датапаков
- `dynamic` — только квесты, созданные командами

Возвращает количество квестов.

**Для чего полезна:** инвентаризация квестов на сервере — отладка, проверка перед обновлением датапака.

**Примеры:**

- *Практичный:* посмотреть все динамические квесты перед обновлением датапака:
  ```
  /quest list dynamic
  ```
- *Креативный:* убедиться, что тестовый квест не утёк в прод-сборку датапака:
  ```
  /quest list static
  ```

---

### `quest list trackedby`

```
/quest list trackedby <player: Selector>
/quest list trackedby <player: Selector> all
/quest list trackedby <player: Selector> active|complete|succeeded|failed|skipped|pinned
```

Выводит квесты, которые отслеживает игрок, с фильтрацией по статусу.

- `all` (по умолчанию) — все отслеживаемые квесты
- `active` — активные (не завершённые)
- `complete` — завершённые (любым способом)
- `succeeded` — завершённые успешно
- `failed` — проваленные
- `skipped` — пропущенные
- `pinned` — закреплённые в HUD

Возвращает количество квестов.

**Для чего полезна:** срез прогресса конкретного игрока — диагностика, поддержка, или нарративная функция.

**Примеры:**

- *Практичный:* диагностика — посмотреть, какие квесты сейчас открыты у игрока:
  ```
  /quest list trackedby PlayerName active
  ```
- *Креативный:* игрок видит свою "летопись поражений" — команда запускается через NPC-свиток:
  ```
  /quest list trackedby @s failed
  ```

---

### `quest list tasks`

```
/quest list tasks <id: Identifier>
/quest list tasks <id: Identifier> all
/quest list tasks <id: Identifier> required
/quest list tasks <id: Identifier> optional
/quest list tasks <id: Identifier> unused
```

Выводит задачи квеста.

- `all` (по умолчанию) — все задачи, сгруппированные по этапам, и неиспользуемые
- `required` — только обязательные (по одной на этап)
- `optional` — только необязательные
- `unused` — задачи, не привязанные ни к одному этапу (созданы, но не добавлены в `stages`)

Возвращает количество задач.

**Для чего полезна:** инспекция структуры этапов квеста при отладке или рефакторинге.

**Примеры:**

- *Практичный:* быстро проверить последовательность обязательных этапов:
  ```
  /quest list tasks story:ch1 required
  ```
- *Креативный:* найти "потерянные" задачи, которые создали, но забыли добавить в этапы:
  ```
  /quest list tasks story:old_quest unused
  ```

## Проверка статусов

---

### `quest test`

```
/quest test <player: Selector> <id: Identifier> active|complete|succeeded|failed|skipped|pinned
/quest test <player: Selector> <id: Identifier> task <taskId: string> active|complete|succeeded|failed|skipped|pinned
```

Проверяет статус квеста или задачи игрока. Возвращает `1` если условие выполнено, `0` если нет.

Удобно использовать в командных блоках для проверки прогресса.

**Для чего полезна:** бинарная проверка статуса — основа автоматизации в командных блоках и функциях.

**Примеры:**

- *Практичный:* проверить перед выдачей награды, что квест успешно завершён:
  ```
  /quest test @s story:ch1 succeeded
  ```
- *Креативный:* если задача-провала выполнена — запустить "плохую концовку" через `setblock` (открыть секретную дверь):
  ```
  /quest test @s story:final_choice task dark_pact succeeded
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Задача не существует (при проверке задачи)

### `quest query`

Возвращает числовой показатель прогресса квеста или задачи. Результат можно сохранить в scoreboard через `/execute store result score`.

**Для чего полезна:** построение скорбордных механик поверх квестов — прогресс-бары, условные триггеры, отображение на табло.

**Прогресс по этапам квеста:**

```
/quest query <player: Selector> <id: Identifier> stages complete|total|percent
```

**Задачи этапа:**

```
/quest query <player: Selector> <id: Identifier> stage active tasks complete|total|percent

/quest query <player: Selector> <id: Identifier> stage <n: int> tasks complete|total|percent
```

`stage active` — задачи текущего активного этапа. `stage <n>` — задачи этапа с номером `n` (нумерация с нуля).

**Прогресс условия конкретной задачи:**

```
/quest query <player: Selector> <id: Identifier> task <taskId: string> success value|target|percent
/quest query <player: Selector> <id: Identifier> task <taskId: string> failure value|target|percent
```

`success` — условие выполнения, `failure` — условие провала. `value` — текущее значение, `target` — целевое, `percent` — процент (0–100). Применимо к условиям с прогресс-баром (например `score` или `all`).

**Метрики `complete`, `total`, `percent`:**

- `complete` — текущее число завершённых единиц (для `stages` — этапов, у которых выполнена обязательная задача; для `tasks` — задач этапа)
- `total` — общее число единиц; работает для квестов в любом состоянии
- `percent` — процент завершения от 0 до 100

> Запросы `complete` и `percent` возвращают ошибку и `0` для завершённых квестов. Запросы `stage active` возвращают ошибку, если у квеста нет активного этапа.

**Примеры:**

- *Практичный:* сохранить число пройдённых этапов в scoreboard для отображения на табло:
  ```
  /execute store result score @s quest_stages run quest query @s example:escape stages complete
  ```
- *Практичный:* обновлять прогресс всех игроков в данже каждый тик (повторяющийся блок):
  ```
  /execute as @a[tag=in_dungeon] store result score @s dungeon_pct run quest query @s story:dungeon stages percent
  ```
- *Креативный:* выдать зелье регенерации, когда сбор трав выполнен ровно наполовину:
  ```
  /execute store result score @s herb_pct run quest query @s story:herbalist task collect_herbs success percent
  /execute if score @s herb_pct matches 50..50 run give @s minecraft:potion{Potion:"minecraft:regeneration"}
  ```

**Ошибки:**
- Квест не существует
- Игрок не отслеживает квест
- Квест завершён (для `complete` и `percent`)
- Нет активного этапа (для `stage active`)
- Индекс этапа вне диапазона (для `stage <n>`)
- Задача не существует (для `task`)

### `/execute if|unless quest` и `/execute if|unless task`

```
/execute if quest <player: Selector> <id: Identifier> active|complete|succeeded|failed|skipped|pinned ...
/execute unless quest <player: Selector> <id: Identifier> active|complete|succeeded|failed|skipped|pinned ...
/execute if task <player: Selector> <id: Identifier> <taskId: string> active|complete|succeeded|failed|skipped|pinned ...
/execute unless task <player: Selector> <id: Identifier> <taskId: string> active|complete|succeeded|failed|skipped|pinned ...
```

Расширение стандартной команды `/execute`. Позволяет использовать статус квеста или задачи как условие в цепочке команд.

**Для чего полезна:** вплетение квестового состояния в обычную командную логику — без отдельных командных блоков, прямо в цепочке `/execute`.

**Примеры:**

```
/execute if quest @s example:escape succeeded run say Квест пройден!
/execute unless quest @s example:escape active run say Квест уже завершён!
/execute if task @s example:escape push_the_lever succeeded run setblock ~ ~ ~ air
```

- *Креативный:* составная логика — приветствие только тем, кто завершил первую главу, но ещё не начал вторую:
  ```
  /execute if quest @s story:ch1 succeeded unless quest @s story:ch2 active run say Добро пожаловать во вторую главу!
  ```

### `quest purge`

```
/quest purge <id: Identifier>
```

Принудительно снимает квест **со всех игроков** (включая оффлайн). Онлайн-игроки получают корректные события (HUD обновится), оффлайн-игроки — тихое удаление из сохранения.

Предназначена для аварийного сброса при редактировании квестов, которые уже были выданы игрокам. Использовать с осторожностью — прогресс всех игроков по квесту будет уничтожен.

Возвращает общее количество затронутых игроков.

**Для чего полезна:** подготовка к структурным правкам квеста (требуется перед `tasks remove` и `quest remove`) или сброс ивента по окончании сезона.

**Примеры:**

- *Практичный:* сбросить квест всем игрокам перед изменением структуры задач:
  ```
  /quest purge story:ch1
  ```
- *Креативный:* по окончании сезонного ивента одной командой обнулить прогресс всех участников, затем удалить квест:
  ```
  /quest purge event:season1_quest
  /quest remove event:season1_quest
  ```

**Ошибки:**
- Квест не существует

| Команда | Описание |
|---------|----------|
