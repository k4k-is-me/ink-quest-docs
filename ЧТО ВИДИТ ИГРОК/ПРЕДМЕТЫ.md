# Предметы

Два физических предмета позволяют взаимодействовать с квестовой системой прямо из инвентаря: книга для просмотра прогресса и свиток для выдачи квестов.

---

## Quest Book (Квестовая книга)

Книга квестов показывает все выданные игроку квесты. Внутри — два раздела: активные и завершённые. При выборе квеста из списка открывается страница с задачами, описаниями и прогрессом. Там же можно закрепить нужную задачу в HUD, не выходя из книги. Как устроен сам интерфейс книги — в [[ИНТЕРФЕЙС ИГРОКА]].

### Как получить

Крафтится в произвольном расположении из трёх предметов:

```
minecraft:book + minecraft:rotten_flesh + minecraft:string
```

### Как открыть

- Нажать клавишу **J** (настраивается в меню управления)
- Нажать ПКМ, держа книгу в руке

### Изменение или отключение рецепта

Датапак может переопределить рецепт книги, разместив файл по тому же пути, что и оригинальный:

```
data/travelcorequesting/recipes/quest_book.json
```

Чтобы **изменить** ингредиенты — напишите свой `crafting_shapeless` или `crafting_shaped` с нужными предметами.

Чтобы **отключить** крафт в выживании — замените ингредиенты на `minecraft:barrier`, который доступен только в режиме креатива:

```json
{
  "type": "minecraft:crafting_shapeless",
  "category": "misc",
  "ingredients": [
    { "item": "minecraft:barrier" }
  ],
  "result": { "item": "travelcorequesting:quest_book", "count": 1 }
}
```

## Quest Scroll (Свиток квеста)

Расходуемый предмет, привязанный к конкретному квесту. При использовании выдаёт игроку этот квест и сразу открывает квестовую книгу на нём. Удобен как внутриигровая "награда" за прохождение другого квеста, торговля с NPC или добыча из сундука.

### Как получить

Рецепта крафта нет. Раздаётся только через:
- Лут-таблицы и сундуки в датапаке
- Торговцев и другие механики датапака
- Команду `/give` с нужным NBT

Стак до 16 штук. Стаки с разными квестами не объединяются.

### Как использовать

Нажать ПКМ, держа свиток. Если квест успешно выдан — стак уменьшается на 1, открывается книга квестов на выданном квесте. Кулдаун 1 секунда защищает от двойного срабатывания.

Если использование невозможно — свиток не расходуется, на action-bar появляется подсказка о причине.

### Создание через команду

```
/give <player> travelcorequesting:quest_scroll{Quest:"namespace:quest_id"}
```

Например, свиток, выдающий квест `story:rescue_blacksmith` игроку:
```
/give @s travelcorequesting:quest_scroll{Quest:"story:rescue_blacksmith"}
```

Поле `Quest` — идентификатор квеста в формате `namespace:id`. Без этого поля свиток будет пустым и не сработает.

## Gamerule `doQuestBookItemCheck`

```
/gamerule doQuestBookItemCheck true|false
```

По умолчанию: **false**. Стандартный gamerule Minecraft — значение сохраняется вместе с миром.

Когда включён, квестовая книга становится физически необходимым предметом: без неё ни клавиша `J`, ни свиток не откроют квесты. Книга должна лежать в инвентаре (любой слот, не обязательно в руке).

| Способ открытия | `false` (по умолчанию) | `true` |
| --- | --- | --- |
| Клавиша **J** | Всегда работает | Нужна книга в инвентаре |
| Quest Scroll | Выдаёт квест и открывает книгу | Без книги — квест не выдаётся, свиток не расходуется |
| ПКМ по книге | Всегда работает | Всегда работает (книга уже в руке) |

> Если игрок использует свиток без книги при включённом gamerule — он видит подсказку на action-bar и не теряет свиток.

Gamerule полезен для карт, где квесты — часть режима выживания и должны органично вписываться в его геймплей. Игрок сначала должен добыть книгу (скрафтить, найти в сундуке или получить как награду), и только после этого сможет читать квесты. До этого квесты могут ему выдаваться и накапливаться, но остаются скрытыми.

### Включение по умолчанию через датапак

Чтобы gamerule включался автоматически при каждой загрузке мира, используйте тег `minecraft:load` в датапаке.

**1.** Создайте или дополните `data/minecraft/tags/functions/load.json`:

```json
{
  "values": ["your_namespace:setup"]
}
```

**2.** Создайте функцию `data/your_namespace/functions/setup.mcfunction`:

```mcfunction
gamerule doQuestBookItemCheck true
```

Замените `your_namespace` на namespace вашего датапака. Функция выполняется при каждом `/reload` — это безопасно, gamerule просто устанавливается в то же значение.
